Basic Git & GitHub

Pengertian Git & GitHub

Git dan GitHub adalah dua alat penting dalam pengembangan perangkat lunak kolaboratif. Mari kita bahas secara singkat apa itu Git dan GitHub serta beberapa konsep dasar yang terkait:

1. Apa itu Git?:
Git adalah sistem kontrol versi yang digunakan untuk mengelola perubahan dalam kode sumber perangkat lunak. Ini memungkinkan pengembang dan tim untuk bekerja bersama pada proyek perangkat lunak, melacak perubahan, dan mengelola versi kode dengan efisien.

2. Konsep Dasar Git:
Repository (Repo): Ini adalah tempat penyimpanan untuk proyek perangkat lunak Kita. Kita memiliki repo lokal (di komputer Kita) dan repo jarak jauh (biasanya di hosting layanan seperti GitHub).
Commit: Ini adalah tindakan menyimpan perubahan kode Kita ke dalam repo. Setiap commit memiliki pesan yang menjelaskan perubahan yang dilakukan.
Branch: Cabang adalah cabang pengembangan dari repo utama. Kita dapat membuat cabang untuk mengembangkan fitur baru tanpa memengaruhi kode di cabang utama.
Merge: Proses menggabungkan perubahan dari satu cabang ke cabang lainnya, seperti menggabungkan fitur yang telah selesai ke dalam cabang utama.
Pull Request: Dalam proyek kolaboratif, PR digunakan untuk meminta peninjauan perubahan dan penggabungan kode ke dalam cabang utama.

3. Apa itu GitHub?:
GitHub adalah platform hosting berbasis web yang digunakan untuk menyimpan, mengelola, dan berkolaborasi dalam pengembangan perangkat lunak dengan menggunakan Git. Ini memberikan alat untuk berbagi kode, melibatkan kontributor, melacak masalah, dan banyak lagi.

4. Konsep Dasar GitHub:
Repository (Repo) GitHub: Ini adalah tempat penyimpanan online untuk proyek Kita di GitHub.
Fork: Kita dapat fork (menyalin) repo orang lain untuk membuat salinan repo di akun GitHub Kita sendiri.
Clone: Clone adalah tindakan menyalin repo dari GitHub ke komputer lokal Kita sehingga Kita dapat bekerja pada kode di komputer Kita.
Pull Request (PR) di GitHub: Ini adalah cara mengusulkan perubahan pada repo seseorang. Setelah Kita membuat perubahan di fork repo Kita, Kita dapat mengajukan PR untuk menggabungkan perubahan tersebut ke repo aslinya.
Issues: GitHub memungkinkan Kita melacak masalah, bug, atau perbaikan yang perlu dilakukan dalam proyek Kita.
